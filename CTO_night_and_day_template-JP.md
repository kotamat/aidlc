## Phase 1: Inception

### Step 1.1: ユーザーストーリーの作成

```
あなたの役割: あなたは熟練したプロダクトマネージャーで、以下のタスクセクションで説明されているシステムを開発するための契約となる、明確に定義されたユーザーストーリーを作成する任務を担っています。

今後の作業の計画を立て、各ステップにチェックボックスを付けてmd形式のファイル（/inception/user_stories_plan.md）に記述してください。いずれかのステップで私の説明が必要な場合は、[Question]タグを付けて質問を追加し、私が回答を記入するための空の[Answer]タグを作成してください。重要な決定を独自に行わないでください。計画が完了したら、私のレビューと承認を求めてください。承認後、同じ計画を一つずつ実行することができます。各ステップが完了したら、計画のチェックボックスに完了印を付けてください。

あなたのタスク: フードデリバリーのシステムを構築します。

/inception/user_stories.mdにユーザーストーリーを記述してください。ユーザーストーリーのみに焦点を当て、それ以外は含めないでください。
```
### (option) UIモックの作成
```

```


### Step 1.2: ユーザーストーリーをユニットにグループ化

```
あなたの役割: あなたは熟練したソフトウェアアーキテクトであり、以下のタスクセクションで説明されているように、ユーザーストーリーを独立して構築できる複数のユニットにグループ分けする任務を担っています。

今後の作業の計画を立て、各ステップにチェックボックスを付けてmd形式のファイル（/inception/units/unit_plan.md）に記述してください。いずれかのステップで私の説明が必要な場合は、[Question]タグを付けて質問を追加し、私が回答を記入するための空の[Answer]タグを作成してください。重要な決定を独自に行わないでください。計画が完了したら、私のレビューと承認を求めてください。承認後、同じ計画を一つずつ実行することができます。各ステップが完了したら、計画のチェックボックスに完了印を付けてください。

あなたのタスク: /inception/user_stories.mdファイルのユーザーストーリーを参照してください。ユーザーストーリーを、独立して構築できる複数のユニットにグループ分けしてください。各ユニットには、単一のチームで構築できる高い凝集性を持つユーザーストーリーが含まれます。ユニット同士は疎結合である必要があります。各ユニットについて、それぞれのユーザーストーリーと受け入れ基準を/inception/units/フォルダ内の個別の.mdファイルに記述してください。まだ技術的なシステム設計は開始しないでください。

各ユニットの統合契約を作成し、各サービスが公開するAPIエンドポイントを記述し、/inception/units/integration_contract.mdファイルにメソッドを定義してください。
```
